# TA-P15-001 Console 工业级规划总纲（2026-03-03）

- Task ID：TA-P15-001
- 阶段：Phase 15
- 状态：DONE
- 负责人角色：TL + Frontend + Backend + QA + DX

## 1. 目标

输出可执行的 Console 工业级规划框架，覆盖功能、架构、多平台、质量与发布策略。

## 2. 功能规划（Functional Scope）

1. 会话域：会话列表、置顶/静音/归档、未读聚合、搜索。
2. 消息域：文本/图片/文件、发送状态、失败重试、撤回展示。
3. 群组域：建群、成员管理、邀请生命周期、群状态视图。
4. 身份与安全：DID 会话态、被撤销 DID 的交互反馈、密钥状态可见性。
5. 设置与账号：节点配置、设备会话、偏好设置。

## 3. 多平台规划（Multi-Platform）

- Web：主端，优先交付。
- PWA：离线缓存、安装能力、通知。
- Desktop（Tauri/Electron 择一）：文件系统能力与本地缓存增强。
- Mobile（React Native/Flutter 择一）：共享协议与数据层，UI 按平台适配。

## 4. 架构规划（Client Architecture）

1. `core-sdk`：协议调用、错误模型、DTO 归一化。
2. `domain`：会话/消息/群组业务状态机。
3. `data`：缓存、离线队列、同步与冲突解决。
4. `ui-shell`：平台无关组件规范 + 主题系统。
5. `platform-adapter`：Web/Desktop/Mobile 差异封装。

## 5. 工业化质量规划

1. 测试：单元、契约、E2E、视觉回归、关键路径压测。
2. 指标：启动时间、首屏可交互、消息发送成功率、拉取延迟。
3. 发布：灰度发布、回滚策略、崩溃采样与诊断。
4. 安全：令牌存储策略、敏感字段脱敏、日志分级。

## 6. Phase 15 里程碑建议

1. M1（架构冻结）：功能域 + IA + 设计系统规范。
2. M2（核心可用）：会话列表 + 聊天窗 + 群组主流程。
3. M3（多平台试点）：Web + PWA 双端一致。
4. M4（工业化收口）：质量门禁通过并进入 Gate。
